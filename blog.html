Local Storage Vs. Session Storage

September 15, 2022 by Tazrian

Session Storage

Whenever a document is loaded in a particular tab in the browser, a unique page session gets created and assigned to that particular tab. This is called Session Storage. The functions of sessionStorage are:

  • It stores data in browser for a session.
  • It saves data in key/value pairs in the browser.
  • It deletes data when the browser is closed.
  • Data is not available for future sessions.
  • It can store data upto 5MB.

The page session is valid only for that particular page loaded in a tab. Opening a page in a new tab or window creates a new session with the value of the top-level browsing context, which differs from how session cookies work. Opening multiple tabs/windows with the same URL creates sessionStorage for each tab/window. Duplicating a tab copies the tab's sessionStorage into the new tab.

Highlights

"While data in localStorage doesn't expire, data in sessionStorage is cleared when the page session ends."

Local Storage

Local Storage is a read-only property of the window interface that allows us to access a Storage object for the Document's origin. The characteristics of localStorage are:

  • It stores data in browser without any expiration date.
  • It saves data in key/value pairs in the browser.
  • It does not delete data when the browser is closed.
  • Data is available for future sessions.
  • It can store data upto 10MB.

localStorage is similar to sessionStorage, except that while localStorage data has no expiration time, sessionStorage data gets cleared when the page session ends â€” that is, when the page is closed. So, in case of Local Storage, the stored data is saved across browser sessions. When data is received in local storage, it is stored as a string.

Global Scope Vs. Block Scope

September 15, 2022 by Tazrian

Scope

The scope is the current context of execution in which values and expressions are visible or can be referenced. If a variable or expression is not in the current scope, it will not be available for use.

Global Scope

Global scope is the default scope for all code running in script mode. It is visible in all other scopes. If any variable or expression is declared in global scope, it can be used anywhere in the document.

Block Scope

The scope created with a pair of curly braces '{ }' is known as block scope. So, if any variable or expression is declared within the block, it cannot be used outside of the block. Function, loops, and conditional statements create block scopes in a document.

How Event Loop Functions?

September 15, 2022 by Tazrian

Event Loop

JavaScript has a runtime model based on an event loop, which is responsible for executing the code, collecting and processing events, and executing queued sub-tasks.
We know that Javascript is a single threaded language. This means it has one call stack and one memory heap. But it can also handle asynchronous calls using event loop. In this case, the codes which requires time to load (like fetching data from another API, or calling setTimeOut() function) are kept in a queue so that they can take time to process the data and then be ready for execution. On the other hand, other codes in the call stack are executed immediately. When the stack becomes empty, then the processed codes are executed one by one from the queue. This process of looping is called event loop.

Stack:

When some code is run, each frame in a stack is created containing references to the function's arguments and local variables. When the first function calls the second, another frame is created and pushed on top of the first one, containing references to the second function's arguments and local variables.
When the second function returns, the top frame element is popped out of the stack. Then the first function returns and the stack becomes empty.

Heap:

Objects are allocated in a heap which is just a name to indicate a large region of memory.

Queue:

JavaScript runtime uses a message queue which contains a list of messages to be processed. Each message has an associated function that gets called to handle the message.
At some point during the event loop, the runtime starts handling the messages on the queue, starting with the oldest one. To do so, the message is removed from the queue and its corresponding function is called with the message as an input parameter. As always, calling a function creates a new stack frame for that function's use.
The processing of functions continues until the stack is once again empty. Then, the event loop will process the next message in the queue (if there is one).

How many ways to get undefined ?

September 15, 2022 by Tazrian

Undefined Value

When any variable or expression is not defined, we get undefined. There are about 8 scenarios where we can get undefined. They are:

  1. Variable that is not initialized
    2. Code:
    let first; console.log(first);
  2. Any function without a return
    3. Code:
    function second(a, b) { const result = a + b; } console.log(second(2, 4));
  3. Parameter that is not passed in any function
    4. Code:
    function third(a, b, c, d) { const total = a + b + c + d; console.log(a, b, c, d); } third(5, 7);
  4. If what to return from a function is not defined
    5. Code:
    function nonNegative(a, b) { if (a < 0 || b < 0) { return; } return a + b; } console.log(nonNegative(3, -9));
  5. A property that doesn't exist on an object
    6. Code:
    const newStudent = { id: 3, name: 'ABC', age: 32 }; console.log(newStudent.age, newStudent.salary);
  6. If accessing array elements out of the index range
    7. Code:
    const array = [21, 32, 34, 32, 45]; console.log(array[0], array[6], array[39]);
  7. If we delete an element from an array using index and then try to access it
    8. Code:
    const array = [21, 32, 34, 32, 45]; delete array[0]; console.log(array[0], array[6], array[39]); console.log(array);
  8. If we assign undefined as a value
    9. Code:
    const shoe = undefined; console.log(shoe);
